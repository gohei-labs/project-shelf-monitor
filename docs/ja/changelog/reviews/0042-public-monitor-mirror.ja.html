Public monitor mirror

推奨MonitorPublic outputDeploy key

判断サマリー

source repository は private のままにし、生成済み monitor site だけを別の public repository gohei-labs/project-shelf-monitor へ publish する。

オーナーに承認してほしいこと

public mirror pattern を承認する。private repository は source of truth のまま、public repository には生成済み monitor output だけを置く。

推奨

進めてよい。private source repository、git history、monitor 以外の files を公開せずに、owner が安定した public monitor URL を持てる。

リスクと軽減策

リスク軽減策
private source history が公開される。source repository は public にしない。生成済み output を別 repository に publish する。
public mirror が source of truth と誤解される。docs と generated README で、generated output だけだと明示する。
広い token が CI に置かれる。gohei-labs/project-shelf-monitor だけに scoped した write deploy key を使う。
secret setup 完了前に publish が失敗する。MONITOR_PUBLISH_SSH_KEY がない場合、workflow は mirror publish を skip し、Actions artifacts は残す。

ここで決めないこと

private source repository の公開、public monitor への認証追加、有料 GitHub plan の選択はここでは決めない。public URL は意図的に公開される。

Terminology Check

用語ここでの使い方
Public mirror生成済み monitor output を受け取る別の public repository。
Source of truthprivate repository 側の source。docs、registry、scripts、review records を含む。
Deploy key1つの repository に紐づく SSH key。ここでは public mirror だけに write できる。

Runtime Boundary Check

runtime は GitHub Actions と PowerShell のまま。新しい helper は CI で実行され、SSH git push を使う。package manager、server runtime、GitHub Pages on public mirror repository 以外の external hosting provider は追加しない。
