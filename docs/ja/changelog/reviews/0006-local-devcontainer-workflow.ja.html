  4. 実機での container execution が通るまで Issue #19 を open のまま残す。

Runtime Position

Docker Desktopの位置づけ

Docker Desktopは、Windows + VS Code Dev Containers の最初の推奨ルートです。ただし、PR #20の要件は製品名ではなく、互換性のあるruntime能力です。

選択肢 位置づけ このPRでの扱い
Docker Desktop + WSL2 Windowsで最も素直なDev Containers導線。Docker公式ライセンス条件の確認が必要。 最初の推奨実装。導入後にpreflightとsmoke testを実行する。
Podman Desktop Docker互換設定により、Docker toolsをPodman engineへ向けられる場合がある。 代替候補。今回のbaselineにはしない。
Rancher Desktop / remote Docker host Docker-compatible workflowを提供できる可能性がある。 代替候補。別Issueで評価する。
Workflow

ローカル検証フロー

PR #20は、実機のcontainer runtime導入前でも「次に何が足りないか」を明確にし、導入後に同じ手順で検証できるようにします。

Step 1

Preflight

scripts/check-local-dev.ps1 で Git、Docker-compatible CLI、daemon、Compose、WSL、shelf path を確認します。

Step 2

Fix Prerequisites

失敗した項目だけを直します。現在は docker CLI がPowerShellから見えていません。

Step 3

Bootstrap

scripts/run-local-dev.ps1 がWindows側のbootstrap/checkを実行し、workspace containerを起動します。

Step 4

Smoke Test

container内でLinux側のbootstrap/checkを実行し、結果をIssue #19に記録します。
