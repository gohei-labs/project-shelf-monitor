モニターのコンテンツ幅を拡大

このレビューは、モニター各ページのコンテンツ最大幅を 1120px から 1440px に引き上げること（広い画面をより活かす）の承認を求めるものです。

Requested Approval

  1. 全モニターページで main { max-width } を 1120px から 1440px に拡大する。これらはダッシュボード／マスター・詳細の UI であり、より広いコンテナは一般的で、特にレビューの読みペイン（iframe）が明確に広くなる。オーナーは 1120px 維持や完全 fluid（全幅）ではなく 1440px を選択。

What Changed

各ジェネレーターの単一値の変更。下記は再生成された HTML。

変更ファイル変更内容
scripts/generate/build-guardrail-monitor.ps1main の max-width を 1120px → 1440px。
scripts/generate/build-application-monitor.ps1main の max-width を 1120px → 1440px。
scripts/generate/build-reviews-page.ps1main の max-width を 1120px → 1440px。
monitor/*.html両言語のページを再生成。

Risk

低リスク。表示のみ。リストページの行は読みやすさを保ち、レビュー文書は iframe 内に独自の最大幅を持つため、コンテナが広がっても本文が不快に伸びることはない。1440px で上限を設けているため、超ワイドなモニターでも全幅にならず余白を残す。

Not Decided Here

  • 完全 fluid（全幅）レイアウト。検討したが採用しなかった。
  • 各ページ内のペイン分割比率。
