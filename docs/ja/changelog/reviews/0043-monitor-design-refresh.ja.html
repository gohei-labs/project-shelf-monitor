Monitor design refresh

推奨MonitorGenerated outputDesign

判断サマリー

生成済み monitor pages を、editorial landing page ではなく、compact な operational dashboard として読める表示へ刷新する。

オーナーに承認してほしいこと

monitor design direction を承認する。guardrail page と review page は、演出的な presentation よりも、scan density、status filtering、generated-source traceability、確実に読めることを優先する。

推奨

進めてよい。新しい surface は monitor という概念に合っている。rule state を素早く点検する場所であり、marketing page や narrative page ではない。

リスクと軽減策

リスク軽減策
初見の読者には monitor が詰まって見える。上部の compact summary blocks を残し、各 guardrail の詳細は modal で読めるようにする。
status filtering で重要な rule を見落とす。default filter は all rules とし、filter ごとに visible count を更新する。
generated HTML が generator と乖離する。monitor / reviews pages は PowerShell generators から再生成し、既存の up-to-date checks で検証する。
mobile text が狭い viewport で切れる。Japanese lead copy は明示的な短い行と mobile wrapping rules を使う。

変更しないこと

guardrail definitions、enforcement behavior、repository visibility、CI policy、public mirror publishing boundary は変更しない。

Terminology Check

用語ここでの使い方
Monitorguardrail と review state を点検するための、生成済み browser-readable surface。
Source of truthprivate source repository にある registry、docs、generator scripts。
Generated outputmonitor/ 配下の HTML files と、それを元に作る Pages site artifact。

Runtime Boundary Check

runtime boundary は変えない。monitor は引き続き既存の PowerShell scripts で生成され、既存 CI で check され、既存の GitHub Pages / public mirror workflow で publish される。package manager、server runtime、Docker dependency、local environment requirement は追加しない。
