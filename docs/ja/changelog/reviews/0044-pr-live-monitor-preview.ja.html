PR live monitor preview

推奨MonitorMobile reviewPublic preview

判断サマリー

pull request ごとの monitor preview を public mirror の pr/<number>/ に publish し、direct links を含む安定した PR comment を追加する。

オーナーに承認してほしいこと

pull request の public・unmerged monitor previews を承認する。preview は generated output のみであり、production monitor root は上書きしない。

推奨

進めてよい。通常の URL は Actions artifact を download するより mobile で確認しやすく、merge 前承認の導線も見えやすい。

リスクと軽減策

リスク軽減策
merge 前の generated output が public になる。公開するのは generated monitor site files だけにし、既存 public mirror boundary の中で pr/<number>/ に置く。
publish credentials が PR 側で変更された scripts に露出する。PR workflow は artifact を upload するだけにする。default branch の trusted な workflow_run workflow が artifact を download し、PR code を実行せずに publish する。
PR preview が production monitor を上書きする。workflow は scoped target path を渡し、publisher は PR preview では mirror root を保持する。
PR に preview comment が重複する。workflow は marker comment が既にある場合、それを更新する。
deploy key が使えず preview publish が実行できない。Actions artifact を fallback preview として残す。
却下された PR previews が public に残る。残るリスクとして記録する。必要なら PR close 時 cleanup を follow-up で追加する。

変更しないこと

private source repository の公開、guardrail source of truth、production root URL への PR preview deploy は変更しない。

Terminology Check

用語ここでの使い方
PR preview1つの pull request の generated monitor output。pr/<number>/ に publish される。
Production monitorpublic mirror root。main からのみ更新される。
Public mirrorGitHub Pages で monitor HTML を host する generated-output repository。

Runtime Boundary Check

runtime は GitHub Actions、PowerShell、git、既存 public mirror repository のまま。preview workflow は GitHub API による PR comment write を追加するが、package manager、server runtime、Docker dependency、新しい hosting provider は追加しない。
