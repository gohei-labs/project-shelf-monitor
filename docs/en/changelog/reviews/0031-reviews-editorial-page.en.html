Reviews Editorial Page

This review asks for approval of step one of moving the control surface to the editorial guardrail-map design and off Docusaurus: a self-hosted, generated "Reviews" index in that same design - a numbered card deck linking each human-review artifact.

Requested Approval

  1. A generator, scripts/build-reviews-page.ps1, that produces playbook/reviews.html and playbook/reviews.ja.html from the artifacts on disk.
  2. A numbered card deck (newest first) with a search box and a top nav between the guardrail page and the reviews page - reachable without Docusaurus.

Risk

Low for this step - it only adds generated pages and a generator, with no change to existing behavior. The larger pivot is deliberately staged: the guardrail-to-reviews cross-link and the full removal of Docusaurus (the docs-portal/ tree, the docs-portal-build guardrail, and the CI Node steps) come as later, focused changes so the working docs site is not torn down before its replacement is wired. The detailed Markdown docs stay as Markdown by owner decision; the editorial site is the control surface only.

Terminology Check

TermMeaning
Control surfaceThe browser-visible pages an owner uses to see and steer the work - here the guardrail map and the reviews index.
Card deckA grid of fixed-width cards; here one card per review, with its number shown prominently.
GeneratorA script that produces the HTML from a source on disk, so the page cannot drift from the artifacts.
Self-hostedServed as plain static files (or opened directly), with no Docusaurus dependency.

Runtime Boundary Check

No new runtime is introduced. The generator is PowerShell, matching the existing guardrail-map builder; it writes static HTML. The pages load Google Fonts over the network (same as the guardrail map) and are otherwise self-contained. This step does not yet remove Docusaurus or change Node usage; that is a later change. The local pre-check, the CI check job, and the Dev Container Bash lane still run the same scripts/check.ps1.
