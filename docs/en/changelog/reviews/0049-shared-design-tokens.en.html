Shared design tokens

This review asks for approval to unify owner-facing surfaces on one canonical token source, starting by bringing the guardrail monitor to the editorial design.

Requested Approval

  1. Make monitor/design-tokens.json the canonical token source for the editorial design (Bone Paper, Warm Ink, Voltage Lime, Space Grotesk + Inter).
  2. Add the shared emitter scripts/generate/editorial-style.ps1 so generators read their colors and fonts from the canonical source instead of each hardcoding its own.
  3. Bring the guardrail monitor to the canonical tokens (editorial palette and fonts; status colors and gate behavior unchanged). Reviews and the application monitor follow in later PRs.

The guardrail monitor's look (palette, fonts) changes. But the status colors (hard gate, conditional, external, etc.) and gate behavior are unchanged — this is presentation only. Tokens now live in one place (design-tokens.json), which structurally prevents future drift. The only view-time external reference is Google Fonts, which degrades to system fonts if blocked.
