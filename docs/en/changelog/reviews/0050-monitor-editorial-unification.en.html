Unify monitor pages on the editorial design

This review asks for approval to bring every monitor page onto one shared editorial design, switch the accent to a "graphite + electric lime" palette, and replace the centered popup with a right-side detail panel.

Requested Approval

  1. Adopt the graphite + electric lime palette as the canonical monitor look: near-white paper, graphite ink, and electric lime (#BCE000) as the single accent used only on the card bars and modal bar. This replaces the interim terracotta. The accent token keys are renamed accent.terracotta/terracottaInk to accent.brand/brandInk.
  2. Unify the monitor pages (guardrails, reviews, development rules, requirements, features) on one layout: single-line page title, a flat full-width list of card rows, and the pillar/kind shown as an inline text tag. The guardrail page no longer groups cards into pillar/category sections.
  3. Replace the centered popup with a right-side detail panel: a persistent two-column layout on wide screens, and a slide-in overlay on narrow screens. The reviews page opens each artifact inline in a right-hand iframe instead of navigating to a separate page.

What Changed

All monitor pages are generator-driven; the HTML files below are regenerated output, not hand edits.

Risk

Generated pages could drift from their sources or be hand-edited. Mitigation: every page is generator-driven, and the guardrail-registry and application-monitor freshness gates in scripts/check.ps1 fail the build if any output drifts; the -Verify runs confirmed all pages are up to date.

The reviews reading pane uses an iframe with a relative ../docs/... path, so it renders only where the site is served from the repository root (local root server or Pages); on narrow screens it falls back to normal navigation. Electric lime is illegible as small text, so it is restricted to bars and markers while all text stays graphite.

Not Decided Here

  • Retrofitting the existing review artifacts 0001-0049, which still use the template's bone-paper / lime palette.
  • Migrating the review-artifact template itself onto the shared design tokens.
  • A dark mode for these surfaces, which the owner has ruled out.
