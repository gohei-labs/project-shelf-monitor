Remove Docusaurus

This review asks for approval to remove the Docusaurus portal and make the control surface a self-hosted set of editorial pages (guardrail map + reviews) in one design language. Detailed Markdown docs stay as Markdown; the editorial pages are the only browser-visible control surface.

Requested Approval

  1. Deletes the docs-portal/ Docusaurus tree and its gate, build step, CI Node steps, and ignore rules.
  2. Cross-links the guardrail map and the reviews index via a shared top nav (completes "reach reviews from the guardrail page").
  3. Adds `scripts/serve-site.ps1` - a localhost-only static server - to view the editorial site.

Risk

Build-then-remove sequencing kept the replacement (the reviews editorial page) in place before deleting the portal, so there is no gap. There is no public or CI-rendered docs site any more - detailed docs are read as Markdown on GitHub or in an editor; if a public site is wanted later it can be generated separately from the editorial sources. The `docs-portal` registry category remains (it groups the advisory browser-control-surface rule) and is harmless.

Terminology Check

Build-then-removeWire the replacement and prove it before deleting the thing it replaces, so no capability gap appears.

Runtime Boundary Check

This change removes a runtime rather than adding one: the Node-based Docusaurus build is gone from the local pre-check and from the CI check job (Node setup deleted). The guardrail suite is now PowerShell-only for the documentation surface. The Dev Container still carries Node as a general feature (unused by the suite now, left in place). The editorial pages are static HTML served locally over http://127.0.0.1 by scripts/serve-site.ps1 (Python's http.server, bound to localhost - never 0.0.0.0, to avoid exposing the repo/`.env` on the LAN). Google Fonts load over the network as before.
