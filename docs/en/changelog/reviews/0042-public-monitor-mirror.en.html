Public Monitor Mirror

RecommendedMonitorPublic outputDeploy key

Decision Summary

Keep the source repository private and publish only the generated monitor site to a separate public repository, gohei-labs/project-shelf-monitor.

What The Owner Needs To Approve

Approve the public mirror pattern: the private repository remains the source of truth; the public repository contains generated monitor output only.

Recommendation

Proceed. This gives the owner a stable public monitor URL without exposing the private source repository, its git history, or non-monitor files.

Risks And Mitigations

RiskMitigation
Private source history becomes public.Do not make the source repository public; publish generated output to a separate repository.
The public mirror is mistaken for source of truth.Docs and generated README label it as generated output only.
A broad token is exposed to CI.Use a write deploy key scoped only to gohei-labs/project-shelf-monitor.
Publishing fails before secret setup is complete.The workflow skips mirror publishing when MONITOR_PUBLISH_SSH_KEY is missing and keeps Actions artifacts available.

What Is Not Decided Here

This does not publish the private source repository, add authentication to the public monitor, or choose a paid GitHub plan. The public URL is intentionally public.

Terminology Check

TermUse here
Public mirrorA separate public repository that receives generated monitor output.
Source of truthThe private repository sources: docs, registry, scripts, and review records.
Deploy keyAn SSH key attached to one repository; here it can write only to the public mirror.

Runtime Boundary Check

The runtime remains GitHub Actions plus PowerShell. The new helper runs in CI, uses SSH git push, and does not add a package manager, server runtime, or external hosting provider beyond GitHub Pages on the public mirror repository.
