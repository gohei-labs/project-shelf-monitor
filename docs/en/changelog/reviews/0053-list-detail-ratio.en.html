Flip the list / detail ratio

This review asks for approval to make the list column narrow and the detail card wide on the non-review monitor pages, and to slim the list rows to fit.

Requested Approval

  1. On the non-review pages (guardrail, development rules, requirements, features), flip the master-detail ratio: the list is now a fixed narrow ~520px column on the left and the detail card fills the remaining width, instead of the previous wide list + 384px card.
  2. To keep the narrower list readable, slim each list row to tag/ID + title + status pill. Secondary fields move into the detail card only: the guardrail "where it runs" (e.g. Local + CI) and the application components / priority. Reviews are unchanged (already list-left + wide reading pane).

Risk

Low; presentation only. The fields removed from the list rows (venue, components, priority) all still appear in the detail card, so no information is lost. Long titles wrap to a second line in the narrow list, which is normal list behaviour.
