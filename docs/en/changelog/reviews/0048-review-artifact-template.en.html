Review artifact template

This review asks for approval to standardize owner-facing review artifacts on a single editorial template, to retire the Runtime Boundary Check, and to auto-generate and verify PR preview links.

Requested Approval

  1. Adopt the editorial layout (near-monochrome with one rationed lime accent, Space Grotesk + Inter) as the official template for owner-facing review artifacts. Metadata (number, status, date) sits in a single top strip.
  2. Fix the section order: Requested Approval → What Changed → Risk → Not Decided Here → Author Checks (Terminology, Primary Sources). Author Checks is a checklist attesting the author verified terminology and primary sources.
  3. Remove the Runtime Boundary Check from the template and disable its CI gate (review-artifact-runtime-boundary / LLM-GR-006). Re-enable enforcement later if it is needed again.
  4. On every PR, have a workflow auto-generate the preview links for the review artifacts it changed and HEAD-verify each returns HTTP 200 before commenting them, so hand-typed dead links cannot happen.

Risk

The visual refresh moves away from the prior blue/amber callout format (0024–0027), but it keeps Requested Approval first and most prominent and limits lime to what the owner must decide, so the decision stays easy to read. Retiring the Runtime Boundary Check weakens the machine check that catches host/tooling assumption mistakes (the PR #36 class of error). It can be re-enabled if needed; the authoritative environment remains the Dev Container plus Docker Compose, and the local pre-check and CI both run the same check.ps1. The only view-time external reference is Google Fonts, which degrades to system fonts if blocked.

Not Decided Here

  • Retrofitting the existing 47 review artifacts (only new artifacts are unified; owner-confirmed, not tracked as an Issue).
  • Reconciling the rules doc (review-artifacts.md) Content Model with the template (separate work; owner-confirmed, not tracked as an Issue).
  • When, if ever, to re-enable the runtime boundary guardrail (decided when the need arises).
