PR Monitor Preview Artifact

RecommendedCIMonitor

Decision Summary

Build the monitor on pull requests and upload it as a downloadable Actions artifact, while keeping GitHub Pages deployment limited to main.

What The Owner Needs To Approve

Approve the preview/deploy split: PRs get a downloadable monitor preview artifact; main gets the stable GitHub Pages deployment.

Recommendation

Proceed. This gives the owner review visibility before merge without letting unmerged branches overwrite the production monitor URL.

Risks And Mitigations

RiskMitigation
Owner cannot inspect the monitor until after merge.PR workflow uploads guardrail-monitor-preview-pr-<number>.
Unmerged branch overwrites the stable Pages monitor.Pages artifact upload and deployment remain restricted to main.
Preview is mistaken for a live hosted site.The workflow summary says to download the artifact and open local HTML files.
Artifact is empty or missing.actions/upload-artifact uses if-no-files-found: error.

What Is Not Decided Here

This does not add per-PR hosted preview URLs. It uses downloadable Actions artifacts only.

Runtime Boundary Check

The runtime remains GitHub Actions plus PowerShell. The workflow adds an artifact upload step and no new package manager, service, or build dependency.
