Docusaurus Documentation Portal

Review artifact for Issue #16. This change creates a browser-visible entry point for inspecting LLM-managed rules and work state.

Requested Review Decision

  1. Adopt docs-portal/ as the Docusaurus source root.
  2. Create the first browser-visible LLM work guardrails pages in Japanese and English.
  3. Build the portal in CI so broken portal changes do not merge.
  4. Promote LLM-GR-004 to Active as the default browser-visible control surface rule.

What This Adds

Portal Root

docs-portal/ contains the Docusaurus site. The untracked root package.json is not used.

Guardrail View

The LLM work guardrails and statuses become available through browser pages.

CI Check

GitHub Actions sets up Node.js and checks that the portal builds.

Core Context Gate

docs/agent-core-context.json defines the fixed reading pack for resumed sessions and future agents.

Out Of Scope

  • No external production deployment in this PR.
  • No full migration of existing Markdown pages in this PR.
  • Do not require npm on the Windows host. The Docusaurus build is validated in CI and should run through Docker / Dev Container workflows.
  • Full cleanup of legacy mojibake in existing Japanese Markdown files is tracked in Issue #37 after owner review identified the tracking gap. Future deferred work requires owner confirmation before a new Issue is created.

Runtime Boundary Check

The mistake found during this PR was caused by overreacting to the local fact that npm was unavailable on the Windows host, without re-checking the existing development environment policy.

BoundaryDecision
Windows hostUsed as the operator machine. It is not the canonical Node/npm runtime for the project.
Docker / Dev ContainerThe reproducible local execution path. Local builds should move through this path.
CIThe required verification path for the Docusaurus build.
GuardrailFuture review artifacts that discuss host/runtime/tool boundaries must include this Runtime Boundary Check.

Handoff Hardening

This PR now adds a fixed Core Context reading gate. The goal is not to detect every possible keyword. The goal is to make every agent start from the same minimum project context before changing code, docs, workflow, GitHub state, or review records.
