Local Pre-Check Evidence

Project Shelf review artifact. Status: Draft for owner review. Date: 2026-06-14.

Japanese version: 0014-local-precheck-evidence.ja.html

Decision Summary

Recommended When local pre-checks run, save their result as JSON evidence under .local/check-runs/.

This makes the verification claim easier to audit. A PR body or Codex Review Record can reference when the checks ran, on which branch, which commands ran, and which commands passed.

Owner Approval

  • Use scripts/save-local-precheck.ps1 as the standard local pre-check evidence entry point before PR review.
  • Store evidence under .local/check-runs/ by default and do not commit it.
  • Reference the evidence path and summary from PR bodies and Codex Review Records.

Not Decided Here

  • Saving evidence as CI artifacts.
  • Signed evidence or tamper-resistant audit records.
  • A large test-reporting system.

Saved Evidence

Field Purpose
timestamp Shows when the checks ran.
branch / head commit Shows which work state was checked.
base ref / changed files Shows which comparison base and changed file set were used for the checks.
working tree state Shows whether uncommitted changes existed during the run.
command list / pass-fail status Shows which checks ran and which checks passed.
short output summary Gives a compact result summary without turning the evidence into a full log archive.

Execution Flow

Prepare PR body draft
->
save-local-precheck.ps1
->
Collect origin/main...HEAD
and local diffs
->
Run checks
->
Write JSON evidence
->
Reference from PR body

Risks And Mitigations

Risk Mitigation
Local evidence Evidence is Git-ignored and local, so it is not an immutable external audit record. Keep this as a small guardrail for making PR-before checks visible. Consider CI artifacts separately if stronger auditability is needed.
Large output Saving full command output can become noisy. Save a short output summary. Use reruns or CI logs for deeper failure investigation.

Terminology Check
