docs 正本駆動の Docusaurus ポータル

Issue #38 のレビュー資料です。Docusaurus をドキュメント本体にせず、docs/ の Markdown をブラウザで見やすく表示するための表示層に戻す判断です。

承認してほしいこと

  1. docs/ をルール、仕様、変更ノートの正本として維持する。
  2. Docusaurus は docs/ の選択された Markdown を直接読む表示層にする。
  3. docs-portal/docs/ に手書きの重複ページを持たない。
  4. CI が Docusaurus のビルド成果物を artifact として残し、オーナーが確認できるようにする。

今回直す問題

以前の状態

docs/ にあるルールや仕様とは別に、docs-portal/docs/ に手書きページを置いていました。

リスク

同じ内容の正本が2つに分かれると、片方だけ更新され、オーナーや次のAIエージェントが古い内容を信じる危険があります。

今回の方針

Docusaurus は docs/ を読み、ブラウザ表示だけを担当します。編集する場所は docs/ に寄せます。

境界の見取り図

領域役割手書きしてよいもの
docs/ 正本。AI と人間が読むルール、仕様、判断、変更ノートを置く。 ルールブック、仕様、変更ノート、レビュー資料、ADR。
docs-portal/ 表示層。Docusaurus が docs/ をブラウザで見せる。 設定、サイドバー、スタイル、入口ページ。
docs-portal/build/ 生成物。CI やローカルで作る確認用の静的サイト。 手書きしない。Git にコミットしない。

成果物の確認方法

確認したいもの見る場所意味
正本の内容 docs/ の Markdown AI エージェントと人間が信頼する元データです。
ブラウザ表示 CI の docs-portal-build artifact Docusaurus でビルドされた静的サイトです。生成物はコミットしません。
レビュー判断 この HTML 今回は「正本と表示層の分離」を承認するための資料です。

Runtime Boundary Check

対象今回の扱い
Windows ホストnpm を必須にしません。ホストの環境整備には広げません。
CINode.js を用意して Docusaurus をビルドし、成果物を artifact として残します。
Docker / Dev Container今後の再現可能なローカル実行経路として維持します。この PR では新しい依存を増やしません。

標準用語チェック

用語この資料での意味

Presentation layer / 表示層正本を読みやすく見せる層。今回は Docusaurus です。
