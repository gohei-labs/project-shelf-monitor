Recommendation

Project Shelf の標準開発環境は Dev Container + Docker Compose にする。Composeはアプリ構成を固定し、Dev Containerは開発者/エージェントが入る作業環境を固定します。

GitHub Codespaces はこのDev Container設定を使ってクラウド上に同じ開発環境を作れます。Copilot coding agent は専用のsetup workflowで依存関係を事前準備します。

Options Considered

OptionStatusReason
Windows native setup Rejected Fast for one PC, but too dependent on local PATH, installed tools, and hidden machine state.
Docker Compose only Partial Good for services, but does not fully define the developer/agent shell environment.
Dev Container + Docker Compose Recommended Defines both services and the actual coding environment. Best fit for agent reproducibility.
GitHub Codespaces Companion Uses the same dev container config for a GitHub-hosted environment.
Copilot setup steps only Companion Useful for Copilot coding agent, but should not be the only local development definition.

Agent Rules

Allowed

  • Run scripts/bootstrap.
  • Run scripts/check.
  • Use Docker Compose services defined in the repo.
  • Update dev environment files through PR review.

Not Allowed

  • Rely on Codex-bundled Python or Node as the project environment.
  • Install random global tools without documenting them.
  • Invent local-only setup steps in chat.
  • Change dependency managers without an ADR or review artifact.

Shelf Mount Rule

Host path H:\ProjectShelf should be mounted into containers as /shelf. The application should use MANAGED_ROOT=/shelf inside containers.

Phase 1 should mount it read-only. Write access requires a later review because it changes data safety risk.
