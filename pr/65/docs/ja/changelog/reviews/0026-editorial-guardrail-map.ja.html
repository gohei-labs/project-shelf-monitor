編集的ガードレール地図

この成果物は、生成されるガードレール地図を、作り込まれた近モノクロの編集的ページ——マストヘッド＋組版的なレジストリ——に作り替えることの承認を求めます。レジストリから完全生成のままで、乖離も引き続きゲートされます。

承認のお願い（Requested Approval）

  1. scripts/build-guardrail-map.ps1 を、playbook/registry.json から編集的ページを生成するよう書き換える：マストヘッド（hero・3本柱・25/10/16 の巨大数字）の下に、全ルールの組版的レジストリ。
  2. 近モノクロ＋配給制の単一アクセント、Google Fonts の Space Grotesk＋Inter、prefers-color-scheme でライト／ダーク、reduced-motion を尊重する読み込み時モーション1回を採用する。
  3. 折りたたみ群と実行場所チップのフィルタを、常時開きのセクション＋単一テキスト検索に置き換える。データと整合は不変で、check-registry.ps1 が引き続き乖離をブロックする。

変更内容（What Changed）

対象変更後の役割
scripts/build-guardrail-map.ps1レジストリから編集的マストヘッド＋レジストリページを生成（見せ方を書き換え、データ導出は維持）。
playbook/guardrail-map.html / .ja.html再生成された出力——手編集しない。
scripts/check-registry.ps1変更なし。引き続き地図とレジストリの一致を検証。

リスク（Risk）

ページは表示時に Google Fonts を参照する。fonts.googleapis.com が遮断されればシステムフォントにフォールバックする（読める）——ビルドやランタイムの依存は追加しない。実行場所・ステータスのチップ絞り込みは単一テキスト検索に置き換えたが、必要なら編集的体裁で戻せる。本変更は見せ方のみで、強制内容は変えない。

用語意味

ランタイム境界チェック（Runtime Boundary Check）

本変更は地図生成器（PowerShell）と生成出力のみの変更で、ビルドやランタイムの依存は導入しない：地図は引き続き静的 HTML としてポータルへコピーされそのまま配信され、ローカル事前チェック・CI の check ジョブ・Dev Container の Bash レーンはいずれも同じ check.ps1 を実行する。

新たな表示時の参照は Google Fonts のスタイルシート（fonts.googleapis.com）のみで、ページを開いた時にブラウザが読み込む。これは任意で、遮断されてもシステムフォントへ優雅に劣化する。権威ある環境は引き続き Dev Container と Docker Compose。
