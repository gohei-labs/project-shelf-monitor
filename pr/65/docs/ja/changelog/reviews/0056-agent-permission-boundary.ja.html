エージェント権限境界: ツール非依存のポリシーをハーネス設定に生成する

この資料は、AIコーディングエージェントが何をしてよいかの単一の正本（agent-policy/policy.json）を設け、Claude Code の権限境界（.claude/settings.json）に生成し、両者を新しいハードゲートで強制することの承認を求めます。

承認のお願い（Requested Approval）

  1. ツール非依存のエージェントポリシーを正本として採用する。 agent-policy/policy.json が deny ルール（破壊的・不可逆コマンド）、保護パス、認証情報の読み取り拒否、egress allowlist、hook 意図、追跡される allow ベースラインを定義する。ハーネス非依存なので、同じポリシーで Claude Code・Codex・Gemini・Cursor を対象にできる。
  2. ハーネス設定は手書きせず生成する。 scripts/generate/build-agent-config.ps1 がポリシーを .claude/settings.json に生成する（本PRは Claude Code の permissions.deny ＋ permissions.allow を出力）。他ハーネスの出力は後で追加し、正本は変えない。
  3. .claude/settings.json を追跡し強制する。 これだけ git 管理に入れる（各自の .claude/settings.local.json は除外のまま）。root-hidden-path ガードレールは .claude を許可するよう更新。新しい agent-permission-boundary ハードゲートが、ポリシーの存在・必須 deny ルール・生成設定の同期を検証する。

リスク（Risk）

.claude/settings.json を追跡すると、広すぎる deny ルールが全員の正当なコマンドをブロックしうる。緩和策: deny ルールは破壊的なものに限定して狭く（force-push、リモートへの hard-reset、force-clean、force-branch-delete、rm -rf / と ~）、allow ベースラインは従来のローカル設定から保持、各自の許可は除外の .claude/settings.local.json に残し、ゲートが同期を検査するのでファイルが正本から無言で乖離できない。pwsh scripts/check.ps1 はパスする。

決めないこと（Not Decided Here）

  • 認証情報の読み取りとネットワーク egress を強制する実行サンドボックス（#62・#63）— ポリシーにデータはあるが本PRでは強制しない。
  • 実行前の決定論的フック、PreToolUse と Stop（#64）— 同じ生成器の後続変更で出力する。
  • Claude Code 以外のハーネス（Codex・Gemini・Cursor）の出力 — 正本を変えずに後で追加。
  • honor venue ルールの advisory への全面再分類 — 本PRは最初のハーネス強制統制を足すのみで、全面再分類は別途。
