Monitor Interface Simplification

RecommendedMonitorGenerated outputDesign

Decision Summary

generated monitor pages から、上部 summary、説明文、search box、filter controls を削除する。

What The Owner Needs To Approve

monitor の方向性を更新する。owner-facing monitor は、top metrics や interactive filtering を持つ dashboard ではなく、grouped records と stable item detail を持つ inspection list とする。

Recommendation

進める。単純化した画面の方が owner feedback に合っており、monitor を直接確認するための画面として焦点が合う。

Risks And Mitigations

RiskMitigation
search を外すことで長い list を探しにくくなる。pillar と category で grouped records を維持し、title、category、status、venue を compact rows に残す。
top counts を外すことで aggregate state が見えにくくなる。aggregate state は source registry から確認できる。owner inspection に必要な場合だけ、装飾ではない形で戻す。
generated HTML が scripts から drift する。monitor output を再生成し、generator verify modes と repository check suite を実行する。
mobile rows が詰まって見える。narrow screens では single-column row layout にし、badges は title の下に wrap できるようにする。

What Is Not Changed

guardrail definitions、review artifact contents、CI policy、publishing credentials、public mirror boundary は変更しない。

Terminology Check

TermUse here
Monitorguardrail と review state を browser で確認するための generated surface。
Inspection listsearch/filter controls なしで、current records を grouped scanning できる compact list。
Source of truthsource repository 内の registry、docs、generator scripts。
Generated outputmonitor/ 配下の checked-in HTML pages と、そこから作られる Pages site artifact。

Runtime Boundary Check

runtime boundary は変わらない。monitor は既存の PowerShell scripts で生成し、既存の guardrails で検証し、既存の GitHub Pages/public mirror workflow で公開する。package manager、server runtime、Docker dependency、local environment requirement は追加しない。
