Application Monitor Baseline

application code を入れる前に、application development を監視する別 monitor を追加する変更です。

Decision Summary

`product/application-monitor.json` を application development rules、requirements、features、architecture assumptions の source of truth にします。default の application monitor page を rules page にし、requirements、features、architecture は別の面で見ます。

What The Owner Needs To Approve

  • structured application monitor source data は、隠しフォルダではなく見える root の `product/` に置く。
  • application monitor を guardrail monitor / review monitor とは別の monitor として扱う。
  • guardrails、reviews、development rules、requirements、features、architecture を sibling top-navigation items として表示する。
  • application-specific development rules を default の application monitor page にする。
  • rules の前に、低情報量な目次だけのページを作らない。
  • requirements、features、architecture / component decisions は別ページに分ける。
  • development rules、requirements、features は compact cards と detail modals で表示する。
  • architecture は component / connection の一覧ではなく、生成された図として表示する。
  • architecture diagram nodes には progress や component-contract status を混ぜない。
  • progress metadata、source links、managed-root summary をこれらの monitor pages に混ぜない。
  • 将来、実装 directory が monitor source と component contract に載らずに増えた場合、check を fail させる。
  • monitor HTML は source から生成し、手で編集しない。

Recommendation

app skeleton を作る前の baseline として承認することを推奨します。開発ルール、要求、機能、architecture diagram を、code が増える前に review できます。

Risks And Mitigations

RiskMitigation
monitor source が古くなる。`check-application-monitor.ps1` が生成 HTML の鮮度を確認し、architecture section に載っていない実装 directory を検出する。
planned component が実装済み code と誤解される。monitor に component status を表示し、directory が存在した後に component contract を必須にする。
application monitor が guardrails と重複する。source と問いが違う。application development rules は `product/application-monitor.json`、general controls は `monitor/registry.json` から見る。

Runtime Boundary Check

architecture diagram には planned な FastAPI / Python と React / Vite / TypeScript components を表示するが、この変更では `apps/api`、`apps/web`、Docker images、dependency installs、runtime commands は作らない。
