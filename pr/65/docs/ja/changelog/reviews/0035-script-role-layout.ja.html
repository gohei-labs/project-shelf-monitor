スクリプト役割別レイアウト

推奨GovernanceTooling

Decision Summary

standard check entrypoints の scripts/check.ps1 と scripts/check.sh は直下に残しつつ、scripts を役割別に分ける。

What The Owner Needs To Approve

script taxonomy を承認する。scripts/guardrails/ は gates、scripts/generate/ は monitor builders、scripts/local/ は local helpers、scripts/setup/ は bootstrap helpers。

Recommendation

進める。top-level に競合する directory を増やさず、guardrail body が見える構造になる。

Risks And Mitigations

RiskMitigation
CI が古い script paths を呼ぶ。workflow calls を更新し、scripts/check.ps1 は shared entrypoint として残す。
registry paths が実ファイルから drift する。monitor/registry.json を更新し、scripts/guardrails/check-registry.ps1 で検証する。
local helper commands が移動する。README と live rule docs に新しい scripts/local/ paths を記録する。

What Is Not Decided Here

local と CI の check suite を分離しない。意図的に scripts/check.ps1 を共有し、local helper utilities だけを分ける。

Terminology Check

TermUse here
Guardrail gatesscripts/guardrails/ 配下の executable validation scripts。
Generatorsmonitor/*.html を書き出す scripts/generate/ 配下の scripts。
Local helpersscripts/local/ 配下の owner/developer convenience scripts。

Runtime Boundary Check

runtime は変えない。PowerShell が guardrail runtime のまま、Bash は PowerShell に委譲し、CI は同じ entrypoint と PR-context review gates を実行し続ける。
