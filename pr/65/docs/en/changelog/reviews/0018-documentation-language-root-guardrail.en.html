Documentation Language Root Guardrail

This review asks for approval to treat docs/en/ and docs/ja/ as the required Markdown documentation roots.

Requested Approval

  1. All new Markdown documentation must be created under both docs/en/ and docs/ja/.
  2. Existing Markdown outside those roots is treated as migration backlog, not accepted structure.
  3. The backlog is explicitly listed in docs/legacy-language-root-allowlist.txt.
  4. CI should reject new Markdown files outside the language roots unless the path is explicitly added to the backlog.

Structure

AreaRule
docs/en/English Markdown documentation.
docs/ja/Japanese Markdown documentation with matching relative paths.
docs/reviews/Bilingual HTML owner-review artifacts, kept separate from Markdown source documents.
Other Markdown under docs/Legacy migration backlog only.

Risk

This change does not move or translate all legacy documents yet. It prevents new drift and makes the remaining migration visible.

Terminology

TermMeaning
Language rootA top-level folder for one documentation language, such as docs/en/ or docs/ja/.
Migration backlogExisting files that are allowed temporarily while being moved into the target structure.
AllowlistAn explicit list of temporary exceptions that CI can check.

Runtime Boundary

This change is a repository documentation and CI check change. It does not require Windows host npm, Docker Desktop, or a running application server.
