Agent permission boundary: a tool-agnostic policy compiled into harness config

This review asks for approval to add a single source of truth for what an AI coding agent may do (agent-policy/policy.json), compile it into the Claude Code permission boundary (.claude/settings.json), and enforce both with a new hard gate.

Requested Approval

  1. Adopt a tool-agnostic agent policy as the source of truth. agent-policy/policy.json defines deny rules (destructive/irreversible commands), protected paths, credential-read denials, an egress allowlist, hook intents, and the tracked allow baseline. It is harness-neutral so the same policy can target Claude Code, Codex, Gemini, and Cursor.
  2. Generate the harness config, do not hand-write it. scripts/generate/build-agent-config.ps1 compiles the policy into .claude/settings.json (this PR emits Claude Code's permissions.deny + permissions.allow). Other harnesses get their own emitter later without changing the source.
  3. Track .claude/settings.json and enforce it. Stop git-ignoring it (per-user .claude/settings.local.json stays ignored); the root-hidden-path guardrail now allows .claude. A new agent-permission-boundary hard gate verifies the policy exists, names the required deny rules, and the generated config is in sync.

Risk

Tracking .claude/settings.json means a deny rule that is too broad could block a legitimate command for everyone. Mitigation: deny rules are narrow and destructive-only (force-push, hard-reset-to-remote, force-clean, force-branch-delete, rm -rf / and ~); the allow baseline is preserved from the prior local config; per-user grants remain in the ignored .claude/settings.local.json; and the gate is in sync-checked so the file cannot silently drift from the policy. pwsh scripts/check.ps1 passes.

Not Decided Here

  • Execution sandboxing with credential-read and network-egress enforcement (#62, #63) — the policy carries that data, but this PR does not enforce it.
  • Deterministic pre-execution hooks, PreToolUse and Stop (#64) — emitted by a later change to the same generator.
  • Emitters for non-Claude-Code harnesses (Codex, Gemini, Cursor) — added later without changing the source of truth.
  • A full relabel of honor-venue rules as advisory — this PR adds the first harness-enforced control; the broader relabel is separate.
