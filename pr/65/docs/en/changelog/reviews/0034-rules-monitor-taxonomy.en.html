Rules And Monitor Taxonomy

RecommendedGovernanceNaming

Decision Summary

Rename the live source-document layer from playbook to rules, and rename the generated browser-visible layer from root playbook to monitor.

What The Owner Needs To Approve

Approve the repository taxonomy: docs/ stores the rule source of truth, monitor/ displays generated views, and scripts/ enforces the guardrails.

Recommendation

Proceed. This matches the owner's mental model: docs act like the law book, while the monitor decides how those rules are visualized.

Risks And Mitigations

RiskMitigation

What Is Not Decided Here

This does not redesign the monitor UI, add a new docs portal, or choose future application architecture. It only fixes the repository's naming and ownership boundaries.
