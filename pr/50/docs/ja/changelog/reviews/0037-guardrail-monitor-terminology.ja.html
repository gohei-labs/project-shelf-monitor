ガードレールモニター用語

推奨統制モニター

判断サマリー

現行用語としての「ガードレール地図」を廃止し、生成されるブラウザ可視ページを「ガードレールモニター」に統一する。

オーナーに承認してほしいこと

monitor vocabulary を承認する。source files が真実を持ち、generated monitor pages がその真実を正確に反映し、人間が監視できるようにする。

推奨

進めてよい。「モニター」は、人間の仕事である project state の監視、drift の発見、注意すべき箇所の判断に合っている。

リスクと軽減策

リスク軽減策
生成 HTML が source からずれる。check-registry.ps1 が monitor/guardrail-monitor*.html と monitor/registry.json の一致を検証する。
過去の記録に "map" が残る。古い review title は履歴として残し、現行 docs、scripts、generated pages、navigation からは外す。
将来の monitor がばらばらに命名される。one source of truth plus generated monitor output という pattern を使う。

Terminology Check

用語ここでの使い方
Monitor人間が監視するための、ブラウザで見える generated view。
Source of truthmonitor content を定義する編集対象の source。ここでは monitor/registry.json。
Generated output再生成すべき monitor/ 配下の HTML。手で編集しない。

Runtime Boundary Check

runtime は変わらない。PowerShell generator path と static HTML output を rename するだけで、local / CI check suite は同じ。
