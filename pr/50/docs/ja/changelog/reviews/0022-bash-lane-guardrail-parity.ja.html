Bash レーンのガードレール同等化

この成果物は、scripts/check.sh を PowerShell へ委譲して Dev Container／CI の Bash レーンに全ガードレールを強制させること、および 2 つのゲートの重複した Bash 版を削除することの承認を求めます。

承認のお願い（Requested Approval）

  1. scripts/check.sh は、手作業で維持する一部ゲートの実行をやめ、Bash レーンの単一の正として pwsh scripts/check.ps1 に委譲する。
  2. Dev Container に ghcr.io/devcontainers/features/powershell で PowerShell 7 を追加し、コンテナ内および devcontainer CI ジョブで委譲が動作するようにする。
  3. 不要になった scripts/check-agent-core-context.sh と scripts/check-docs-portal.sh を削除する（PowerShell 版が check.ps1 内で既に実行される）。
  4. pwsh が無い場合、Bash レーンはチェック未実行で通過せず fail-closed で失敗する。

変更内容（What Changed）

対象変更後の役割
scripts/check.sh薄い Bash 入口。--quick を引き継ぎつつ pwsh scripts/check.ps1 を exec する。
scripts/check.ps1全レーン（Windows ホスト、PowerShell CI ジョブ、Dev Container、Bash）が実行する単一のガードレール一式。
.devcontainer/devcontainer.jsonPowerShell フィーチャを追加し、コンテナ内に pwsh を用意する。
削除した .sh 補助スクリプトcheck-agent-core-context.sh と check-docs-portal.sh — .ps1 版に置き換え済み。

リスク（Risk）

本変更前、devcontainer CI ジョブはガードレールスクリプトを 2 つ（core-context と docs-portal）しか実行しておらず、他のゲートで落ちる変更でも通過し得た。本変更後は Bash レーンが scripts/check.ps1 一式（テキストエンコーディング、二言語対、言語ルートと内容、ドキュメント命名、レジストリ整合、秘密漏れ、実装前の決定、レビュー成果物のランタイム境界など、Windows ホストや CI の check ジョブと同じゲート）を実行する。PR 文脈に依存する 4 つのレビューゲートと devcontainer-build ジョブは、従来どおり別の CI ジョブのまま変更しない。主な残存コストは、devcontainer ジョブがドキュメントポータルを 2 回（quick→full）ビルドすることで、これは性能上のコストであり強制の穴ではない。

用語チェック（Terminology Check / 標準用語チェック）

用語意味
ガードレール（Guardrail）プロンプトではなく、ビルドを失敗させるスクリプトで強制される規則。
チェックレーン（Check lane）ガードレール一式が走る 1 つの環境：Windows ホスト、PowerShell CI ジョブ、または Dev Container の Bash。
単一の正（Single source of truth）全レーンが呼び出す唯一の正本実装（scripts/check.ps1）。レーン間の乖離（drift）を防ぐ。
fail-closedツールが実行できない場合（ここでは pwsh 不在）、通過させずに失敗させること。

ランタイム境界チェック（Runtime Boundary Check）

本変更はランタイム境界そのものに関わる。権威ある環境は引き続き Dev Container と Docker Compose であり、ホストにたまたま入っているシェルではない。PowerShell 7（pwsh）は、ホストに前提として存在することを仮定するのではなく、公式の PowerShell フィーチャ経由で導入される、明示された Dev Container の依存になった。

Windows ホストでは既に pwsh scripts/check.ps1 で一式が走る。Linux・macOS・CI では bash scripts/check.sh 経由で走り、これが同じ PowerShell 一式を exec する。Windows 専用ツールは導入しない。ドキュメントポータルのビルドには、Node フィーチャが既に提供する Node と npm が引き続き必要。
