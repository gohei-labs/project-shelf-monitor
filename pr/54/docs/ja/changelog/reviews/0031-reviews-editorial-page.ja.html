レビューの editorial ページ

この成果物は、コントロール面を editorial なガードレール地図のデザインに寄せ、Docusaurus から離れる第一歩——同じデザインの自前ホスト「レビュー」索引（各人間レビュー成果物にリンクする番号付きカードデッキ）——の承認を求める。

承認のお願い（Requested Approval）

  1. 生成器 scripts/build-reviews-page.ps1。ディスク上の成果物から playbook/reviews.html と playbook/reviews.ja.html を生成。
  2. 番号付きカードデッキ（新しい順）+ 検索ボックス + ガードレールページとの相互ナビ。Docusaurus 不要で到達できる。

リスク（Risk）

本ステップのリスクは低い——生成器と生成ページを足すだけで、既存挙動は変えない。大きな転換は意図的に段階化する：ガードレール→レビューの相互リンクと、Docusaurus（docs-portal/ ツリー、docs-portal-build ガードレール、CI の Node 手順）の完全撤去は、置換が整う前に動いているドキュメントサイトを壊さないよう、後続の集中変更とする。詳細 Markdown はオーナー判断で Markdown のまま。editorial サイトはコントロール面のみ。

ランタイム境界チェック（Runtime Boundary Check）

新たなランタイムは導入しない。生成器は既存のガードレール地図ビルダーと同じ PowerShell で、静的 HTML を書き出す。ページは（ガードレール地図と同様）Google Fonts をネットワークから読むほかは自己完結。本ステップではまだ Docusaurus を撤去せず、Node の利用も変えない（後続変更）。ローカル事前チェック、CI の check ジョブ、Dev Container の Bash レーンは引き続き同じ scripts/check.ps1 を実行する。
