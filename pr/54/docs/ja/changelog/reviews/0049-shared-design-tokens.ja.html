共有デザイントークン

この資料は、オーナー向けサーフェスのデザインを単一の正本トークンに統一する第一歩として、guardrail monitor を編集的デザインに揃えることの承認を求めます。

承認のお願い（Requested Approval）

  1. monitor/design-tokens.json を、編集的デザイン（Bone Paper・Warm Ink・Voltage Lime・Space Grotesk＋Inter）の正本トークンにする。
  2. 共通エミッタ scripts/generate/editorial-style.ps1 を新設し、ジェネレータが色とフォントを正本トークンから読む形にする（各サーフェスが個別に色を持たない）。
  3. guardrail monitor を正本トークンに寄せる（パレットとフォントを編集的に統一。ステータス色とゲートの機能は不変）。reviews / application monitor は次PRで順次。

変更内容（What Changed）

承認のお願いを実現する、具体的な変更ファイルの一覧です。

変更ファイル変更内容
monitor/design-tokens.json編集的トークンの正本を新設（palette・accent・fonts・pillarAccents・status）。
scripts/generate/editorial-style.ps1トークンを読む共通エミッタを新設（フォントリンク・pillar accent も提供）。

リスク（Risk）

guardrail monitor の見た目（パレット・フォント）が変わる。ただしステータス色（強制ゲート・条件付き・外部など）とゲートの機能は変えず、表示のみの変更。トークンは1か所（design-tokens.json）に集約したので、以降の揺れを構造的に防げる。外部参照は表示時の Google Fonts のみで、遮断時はシステムフォントへ劣化する。

決めないこと（Not Decided Here）

  • reviews / application monitor を正本トークンへ寄せる作業（次PRで順次）。
  • 既存 47 本のレビュー資料の遡及統一（別途）。
  • レビュー資料（自己完結HTML）自体を正本トークンから生成する仕組み（将来。現在はインライン据え置き）。
  • モニターの変数名を正本名へ完全統一する作業（今回は既存名を保持し、値だけ正本化）。
