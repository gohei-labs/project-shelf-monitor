Human Review Issue #25 GitHub Operations

GitHub token運用を迷子にならない形で固定する

PR #24で、正しいfine-grained PATは存在していたのに、通常のgh認証を見に行って403になりました。 このPRでは、次のLLMや人間が同じ迷子にならないように、tokenの置き場所、権限、使い方、失敗時の見分け方を文書化します。

承認してほしいこと
  1. .env の GH_TOKEN をrepository-local credential sourceとして扱う。
  2. fine-grained PATの期待権限をspecに固定する。
  3. .local helperの使い方を正式docsから参照できるようにする。
  4. gh pr checksが読めない場合はgh run listをCI確認のfallbackにする。

Decision

採用する運用

項目 決定
Token location .env の GH_TOKEN。値は表示しない。
Command shape powershell -NoProfile -ExecutionPolicy Bypass -File .local\gh-project-shelf.ps1 ... を標準形にする。
Git commands .local\git-project-shelf.ps1 経由でpull/pushする。
CI status gh pr checks がtoken制約で失敗したら gh run list を使う。
Durable source of truth docs/specs/github-token-management.md と日本語版。
