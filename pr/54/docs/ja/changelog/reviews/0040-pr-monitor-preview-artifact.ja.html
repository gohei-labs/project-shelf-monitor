PR モニター preview artifact

推奨CIMonitor

判断サマリー

pull requests で monitor を build し、download 可能な Actions artifact として upload する。一方、GitHub Pages deployment は main のみに限定する。

オーナーに承認してほしいこと

preview/deploy split を承認する。PR では download 可能な monitor preview artifact を使い、main では stable な GitHub Pages deployment を使う。

推奨

進めてよい。merge 前の review visibility を確保しつつ、unmerged branches が production monitor URL を上書きしない。

リスクと軽減策

リスク軽減策
owner が merge 後まで monitor を確認できない。PR workflow が guardrail-monitor-preview-pr-<number> を upload する。
unmerged branch が stable Pages monitor を上書きする。Pages artifact upload と deployment は main に限定する。
preview が live hosted site と誤解される。workflow summary で artifact を download し、local HTML files を開くよう案内する。
artifact が空または欠落する。actions/upload-artifact で if-no-files-found: error を使う。

ここで決めないこと

PR ごとの hosted preview URL は追加しない。download 可能な Actions artifacts だけを使う。

Terminology Check

用語ここでの使い方
Actions artifactGitHub Actions run に紐づく download 可能な file bundle。
GitHub Pages deploymentmain からだけ生成される stable hosted monitor。
Preview artifactmerge 前 review に使う pull-request monitor bundle。

Runtime Boundary Check

runtime は GitHub Actions と PowerShell のまま。workflow は artifact upload step を追加するだけで、新しい package manager、service、build dependency は導入しない。
