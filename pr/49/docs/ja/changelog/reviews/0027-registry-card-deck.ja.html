レジストリ・カードデッキ

この成果物は、ガードレール・レジストリを、一定幅のカードのデッキ＋クリックで開く詳細モーダルとして見せること（縦長の一覧を置き換え）の承認を求めます。レジストリから生成のままで、乖離も引き続きゲートされます。

承認のお願い（Requested Approval）

  1. 各ガードレールを一定幅のカード（従来どおり柱→カテゴリでグループ化）として描画し、柱ごとのアクセントを付ける：ガードレール＝lime、可視化＝cyan、人間オーケストレーション＝violet。塗りマーカー＝強制、白丸＝それ以外。
  2. カードをクリックすると、中央の詳細モーダル（規則・強制元・実行場所・状態・備考）がやわらかな拡大＋フェードで1つだけ開く。背景・閉じるボタン・Escape で閉じる。
  3. マストヘッドは不変、検索も維持（カードを絞り込む）。データと整合は不変で、check-registry.ps1 が引き続き乖離をブロックする。

リスク（Risk）

ダークなカードは意図的に明るい編集的地の上に置く。ダークホスト時はカード面を地より少し明るくして区別を保つ。モーダルは fixed 配置で、単体ページでもポータルでも正しく動く。既存の Google Fonts 参照以外にビルドやランタイムの依存は追加せず、本変更は見せ方のみで強制内容は変えない。

ランタイム境界チェック（Runtime Boundary Check）

本変更は地図生成器（PowerShell）と生成出力のみの変更。地図は引き続き静的 HTML としてポータルへコピーされそのまま配信され、ローカル事前チェック・CI の check ジョブ・Dev Container の Bash レーンはいずれも同じ check.ps1 を実行する。ビルドやランタイムの依存は追加しない。

表示時の参照は引き続き Google Fonts のみで、ページを開いた時にブラウザが読み込み、遮断時はシステムフォントへ優雅に劣化する。権威ある環境は引き続き Dev Container と Docker Compose。
