CI でのレジストリ・トグル整合

この成果物は、レビュー 0024 で「現状の制限」として挙げた穴の修正の承認を求める。CI の human-review-artifact ジョブは 4 つのレビューゲートを直接実行していたため、レジストリの enabled トグルがそこへ届いていなかった。これらのガードレールを無効化しても CI では黙って無視され、記録された disabledReason が偽りの安心を与えていた。本修正で、すべての強制レーンが同じトグルを尊重する。

承認のお願い（Requested Approval）

  1. CI が直接呼ぶ 4 ゲートがレジストリを読み、無効時はスキップするので、それらにとってもレジストリが唯一の on/off スイッチになる。
  2. review-artifact-runtime-boundary は 2 レーンで走り、無効化時に split-brain（分裂）していた。両レーンが一致するようになった。
  3. check-registry.ps1 は、CI に配線されたスクリプトゲートがトグルを忘れると失敗するので、迂回が黙って再発しない。

変更内容（What Changed）

対象変更後の役割
scripts/guardrail-lib.ps1新しい共有ヘルパー。レジストリを読む唯一の Test-GuardrailEnabled（fail-closed）。
scripts/check.ps1自前の実装をやめてヘルパーを読み込む——トグル実装は 1 つ。
4 つのレビューゲートヘルパーを dot-source し、無効時は早期に exit。CI でもトグルを尊重。
scripts/check-registry.ps1CI 配線のスクリプトゲートがトグルを参照しているか検証。
scripts/check-spec-before-impl.ps1guardrail-lib.ps1 をガードレールツールとして除外。

リスク（Risk）

本変更はガードレール・ツールの変更であり、プロダクトではない。今日は全ガードレールが有効なので挙動は不変。残る制限は devcontainer-build ガードレールで、これはゲートスクリプトを持たない CI ジョブ（Dev Container をビルドする）であるため、ジョブ単位のトグルはまだ未配線で、スクリプトゲートのみを対象とする新 assertion はこれを検出しない。別の後続作業として記録する。

用語チェック（Terminology Check / 標準用語チェック）

用語意味
強制レーン（enforcement lane）ガードレールが実際に走る経路：ローカル事前チェック、CI の check ジョブ、CI の human-review-artifact ジョブ、Dev Container の Bash レーン。
トグル迂回（toggle bypass）レジストリの enabled を読まずにゲートを走らせる強制レーン。無効化したガードレールがそこではまだ走る。
split-brain（分裂）ガードレールの on/off について 2 レーンが食い違うこと——ここでは check ジョブが無効ゲートをスキップする一方、CI が強制し続けていた。
fail-closedエラー時（レジストリが読めない時）は、飛ばさず全ガードレールを有効扱いするのが安全側の既定。

ランタイム境界チェック（Runtime Boundary Check）

新たなランタイムは導入しない。本変更は PowerShell のヘルパーを 1 ファイル足し、既存のゲートスクリプトにレジストリを読ませるだけ。ローカル事前チェック（PowerShell）、CI の check ジョブ、Dev Container の Bash レーンは、いずれも同じ scripts/check.ps1 を実行する。CI の human-review-artifact ジョブは 4 ゲートを Node 不要の PowerShell で直接実行する。本変更後はそれらすべてのレーンが同じレジストリのトグルを参照する。新しい依存（Node・npm・Docker・パッケージマネージャ）は追加しない。
