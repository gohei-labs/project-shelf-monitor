Registry Card Deck

This review asks for approval to show the guardrail registry as a deck of fixed-width cards with a click-to-open detail modal, replacing the tall vertical list - still generated from the registry and still gated against drift.

Requested Approval

  1. Render each guardrail as a fixed-width card (grouped by pillar then category) with a per-pillar accent: lime for Guardrails, cyan for Visualization, violet for Human orchestration. A filled marker means enforced, a hollow ring means not.
  2. Clicking a card opens one centered detail modal (rule, enforced-by, where it runs, status, note) with a soft scale-and-fade; backdrop, close button, or Escape dismisses it.
  3. Keep the masthead unchanged and the search working (it now filters cards). The data and integrity are unchanged - check-registry.ps1 still blocks registry/map drift.

Risk

The dark cards sit on the light editorial ground by design; in dark host mode the card face is slightly lighter than the page so they stay distinct. The modal uses fixed positioning, correct for the standalone page and the portal. No build or runtime dependency is added beyond the existing Google Fonts reference, and the change is presentation only - it does not alter what is enforced.

Terminology Check

TermMeaning
Card deckThe registry rendered as a grid of fixed-width card tiles, one per guardrail.
Detail modalA single centered overlay panel that shows a card details on click.
Per-pillar accentA cohesive cool color (lime / cyan / violet) that marks which pillar a card belongs to.
Enforced markerA filled square (enforced) versus a hollow ring (conditional, advisory, or external).

Runtime Boundary Check

This change is to the map generator (PowerShell) and its generated output only. The maps are still static HTML copied into the docs portal and served as-is; the local pre-check, the CI check job, and the Dev Container Bash lane all run the same check.ps1. No build or runtime dependency is added.

The only view-time reference remains the Google Fonts stylesheet, loaded by the browser when the page is opened, with graceful fallback to system fonts. The authoritative environment remains the Dev Container plus Docker Compose.
