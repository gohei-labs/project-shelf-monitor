Tracked GitHub Token Wrappers

RecommendedCredential workflowLocal operations

Decision Summary

Move reproducible GitHub command wrappers into tracked scripts/local/, while keeping the token itself only in ignored .env or the process environment.

What The Owner Needs To Approve

Approve the rule that .local/ may hold disposable local artifacts, but not the only copy of an operational script needed for push, PR, or CI inspection.

Recommendation

Proceed. The previous design made the command path depend on an ignored directory. Tracking the wrappers keeps cleanup safe without committing secrets.

Risks And Mitigations

RiskMitigation
The token is accidentally printed.The wrappers load GH_TOKEN into process environment and do not echo its value.
Stored Git credentials override the intended token.The Git wrapper disables credential.helper for the invocation and uses askpass.
The original .local/ helper cannot be recovered.The reusable wrapper logic is now tracked; only the secret token must be restored separately.
A broad token is used to recover quickly.The rule still requires the smallest repository-scoped token that supports the task.

What Is Not Decided Here

This does not create, broaden, or expose a GitHub token. It only makes the command wrapper reproducible.

Terminology Check

TermUse here
Credential sourceThe place the command reads the token from, preferably .env or process environment.
Wrapper scriptA tracked command entry point that loads credentials without storing them.
AskpassA Git mechanism for answering username and password prompts without interactive typing.

Runtime Boundary Check

The wrappers are PowerShell local helper commands. They do not change CI runtime, GitHub Actions configuration, or the Dev Container contract.
