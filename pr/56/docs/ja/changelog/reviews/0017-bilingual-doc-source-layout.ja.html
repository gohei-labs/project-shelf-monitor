docs Markdown 表示

Issue #40 / PR #41 の修正レビューです。Docusaurus は独自ページを主役にせず、docs/ 配下の Markdown をブラウザで表示するサイトにします。

承認してほしいこと

  1. Docusaurus の対象を docs/ 全体の Markdown にする。
  2. 手で選んだ数ページだけを表示する構成をやめる。
  3. トップページの独自説明をやめ、/docs/ に移動する。
  4. ページ下部の Previous / Next ナビゲーションを非表示にする。
  5. 画面上の名称を Project Shelf Docs にし、独自の管理ページに見える表現を避ける。

変更内容

対象変更
Docusaurus docs path../docs を直接読む。
Sidebarautogenerated を使い、docs/ 配下の Markdown を自動表示する。
Home page独自トップページをやめ、/docs/ へ移動する。
PaginationPrevious / Next を CSS で非表示にする。

リスクと対策

リスク: docs/ 全体を表示すると、古い資料や作業途中の資料も見える可能性があります。

対策: まずは「全部見える」ことを優先し、整理や非表示ルールは別途オーナー確認してから決めます。

標準用語チェック

用語この資料での意味

実行環境境界チェック

Docusaurus build は CI と Docker / Dev Container 前提の開発環境で検証します。Windows ホスト上の一時的な Node/npm 状態を、プロジェクトの開発環境要件にはしません。

情報源この判断で支える内容
