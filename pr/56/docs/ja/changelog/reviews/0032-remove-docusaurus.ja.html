Docusaurus の撤去

この成果物は、Docusaurus ポータルを撤去し、コントロール面を自前ホストの editorial ページ群（ガードレール地図 + レビュー）にひとつのデザイン言語で統一することの承認を求める。詳細 Markdown は Markdown のまま。editorial ページが唯一のブラウザ可視な統制面。

承認のお願い（Requested Approval）

  1. docs-portal/ の Docusaurus 一式と、そのゲート・ビルド手順・CI の Node 手順・無視ルールを削除。
  2. ガードレール地図とレビュー索引を共通トップナビで相互リンク（「ガードレールページからレビューに行ける」を完成）。
  3. scripts/serve-site.ps1（localhost 限定の静的サーバ）を追加し、editorial サイトを閲覧可能に。

変更内容（What Changed）

対象変更
docs-portal/、scripts/check-docs-portal.ps1削除。
playbook/registry.jsondocs-portal-build を除去（24 ガードレールに）。地図を再生成。
scripts/check.ps1、check-spec-before-impl.ps1docs-portal のビルド節とコードパスを削除。
.github/workflows/ci.ymlcheck ジョブから Node セットアップ + docs-portal アーティファクトを削除。
scripts/build-guardrail-map.ps1ガードレール / レビューのトップナビを追加。
scripts/serve-site.ps1新しい localhost 静的サーバ。
documentation-system・index・引継ぎブリーフ（両言語）Docusaurus ではなく editorial サイトを説明。

リスク（Risk）

「作ってから消す」順序で、撤去前に置換（レビュー editorial ページ）を用意したので空白期間は無い。公開／CI レンダリングのドキュメントサイトはもう無く、詳細ドキュメントは GitHub やエディタで Markdown として読む。将来公開サイトが必要なら editorial ソースから別途生成できる。docs-portal というレジストリのカテゴリは残る（advisory の browser-control-surface をまとめる）が無害。

用語チェック（Terminology Check / 標準用語チェック）

用語意味
Docusaurusこれまでブラウザ向けドキュメントポータルに使っていた Node の静的サイト生成基盤。今回撤去。
editorial コントロールサイトブラウザ可視の統制面である、自前ホストの生成 HTML ページ（ガードレール地図 + レビュー）。
コントロール面オーナーがブラウザで作業を見て舵を取るための面。
作ってから消す置換を用意し動作確認してから、置き換える対象を消す。能力の空白を作らない。

ランタイム境界チェック（Runtime Boundary Check）

本変更はランタイムを足すのではなく外す：Node ベースの Docusaurus ビルドが、ローカル事前チェックと CI の check ジョブから無くなる（Node セットアップ削除）。ドキュメント面のガードレールスイートは PowerShell のみになる。Dev Container は一般機能として Node を依然保持（今はスイート未使用だが残置）。editorial ページは静的 HTML で、scripts/serve-site.ps1（Python の http.server、127.0.0.1 バインド——.env を含むリポジトリを LAN に晒さないため 0.0.0.0 にはしない）が http://127.0.0.1 でローカル配信する。Google Fonts は従来どおりネットワークから読む。
