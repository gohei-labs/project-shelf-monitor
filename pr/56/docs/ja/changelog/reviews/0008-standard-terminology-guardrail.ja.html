Human Review Issue #23 Terminology Guardrail

内輪の言葉を標準用語へ寄せる仕組みを入れる

レビュー資料が独自の言い回しに流れると、人間もAIエージェントも判断しにくくなります。 重要な概念を説明するHTMLレビュー資料には、標準用語を確認する欄を必須にします。

承認してほしいこと
  1. 標準用語を優先するガードレールを Active にする。
  2. 変更された review HTML に Terminology Check 欄を必須にする。
  3. PR template に標準用語確認のチェック項目を追加する。
Problem

何が外れたか

PR #20 のレビュー資料では、実質的には preflight check や smoke test の話をしているのに、 説明が独自の言い回しに寄り、業界で広く使われる概念として整理されていませんでした。

Internal Wording

独自表現が混ざる

その場の会話に引っ張られた言葉のまま資料化すると、あとから読んだ人が理解しにくくなります。

Review Miss

レビューで見逃す

内容の方向性だけを見ると、用語のずれを品質問題として拾えないことがあります。

CI Guardrail

確認欄を強制する

CIで用語確認欄の存在を確認し、人間が見える形で標準用語への対応付けを求めます。

Terminology Check

標準用語チェック

この資料で使う標準用語と、避けたい曖昧な表現の対応です。

標準用語 意味 避けたい表現
standard terminology 業界や技術コミュニティで広く使われる用語。レビュー資料ではまずこちらを使います。 内輪の言葉、独自の言い回し
terminology check 重要概念が標準用語で説明されているかを確認する欄。 なんとなく分かる説明、雰囲気で通じる説明
review artifact オーナーが判断しやすいように作る、人間向けのレビュー資料。 資料、HTML、説明ページだけで済ませる表現
guardrail 同じミスを防ぐため、作業手順やCIに組み込む制約。 気をつける、注意する
Mechanism

どう強制するか

層 仕組み 効き方
CI scripts/check-review-artifact-terminology.ps1 変更された docs/reviews/*.html に用語確認欄がなければ失敗します。
PR template Human Review Artifact の checkbox PR 作成時に、作業者が標準用語チェックを意識できます。
Spec / Registry LLM-GR-001 を Active 化 LLM-assisted work 全体の標準ルールとして扱います。
Limits

限界

CIは「確認欄が存在するか」を見ます。用語選びの質そのものは、人間レビューとCodex Review Recordで確認します。

できないこと
  • すべての用語選択が正しいかを完全に自動判定する。
  • 分野ごとの最適用語をCIだけで決める。
  • 読みやすさの最終判断を人間から奪う。
できること
  • 重要な概念を説明する場面で、用語確認を省略しにくくする。
  • 標準用語と独自表現の対応をレビュー対象として表に出す。
  • 次のPR #20修正で、同じ観点を必ず確認する入口を作る。
