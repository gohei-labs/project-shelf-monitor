Widen the monitor content width

This review asks for approval to raise the monitor pages' content max-width from 1120px to 1440px, so the surfaces use more of a wide screen.

Requested Approval

  1. Increase main { max-width } from 1120px to 1440px on every monitor page. These surfaces are dashboard / master-detail UIs where a wider container is conventional, and the reviews reading-pane iframe in particular gains noticeably more room. The owner chose 1440px over keeping 1120px or going fully fluid (full-bleed).

What Changed

A single value in each generator; the regenerated HTML follows.

Changed fileWhat changed

monitor/*.htmlRegenerated pages in both languages.

Risk

Low; presentation only. The list pages keep readable rows, and the reviews document has its own internal max-width inside the iframe, so the wider container does not stretch body text uncomfortably. Still bounded (1440px), so very wide monitors keep a margin rather than going full-bleed.

Not Decided Here

  • A fully fluid (full-bleed) layout, which was considered and not chosen.
  • The per-pane split ratios within each page.
