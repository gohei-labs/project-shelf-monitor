開発ルール

APP-RULE-001

アプリケーションモニター正本を先に更新

アプリケーションの要求、開発ルール、コンポーネント、接続を変える場合は、生成 HTML より先に product/application-monitor.json を更新する。

根拠: check-application-monitor.ps1 が生成済み monitor の鮮度を確認する。

有効

統制: 強制ゲート

APP-RULE-002

実装ディレクトリは宣言必須

apps/、packages/、services/ 配下の実装ディレクトリは、application monitor の正本と project-components.json の両方に載せる。

根拠: guardrail が実ディレクトリ、monitor 正本、component contract を照合する。

有効

統制: 強制ゲート

APP-RULE-003

実装前に要求のつながりを明示

product behavior は、実装前に requirement、feature、responsible component のつながりを示す。

根拠: requirements page と architecture page が、requirement、feature、component のつながりを別々に表示する。

有効

統制: 運用合意

APP-RULE-004

Phase 1 は読み取り専用

Phase 1 では、オーナーファイルの移動、削除、コピー、インポート、書き換えを行う UI control や API behavior を追加しない。

根拠: 読み取り専用の挙動を application development rule と product requirement の両方で追跡する。

有効

統制: 運用合意

APP-RULE-005

アーキテクチャ変更は専用ページで見る

component、folder、stack、connection の判断は rules や requirements page に混ぜず、architecture page に置く。

根拠: generator が dedicated architecture page を生成する。

有効

統制: 運用合意
