GitHub トークン wrapper の tracked 化

推奨Credential workflowLocal operations

判断サマリー

再現可能な GitHub command wrappers を tracked な scripts/local/ に移し、token 自体は ignored な .env または process environment にだけ置く。

オーナーに承認してほしいこと

.local/ は disposable local artifacts を置く場所であり、push、PR、CI inspection に必要な operational script の唯一の copy を置かない、という rule を承認する。

推奨

進めてよい。以前の設計では command path が ignored directory に依存していた。wrapper を tracked にすることで、secrets を commit せずに cleanup に強くできる。

リスクと軽減策

リスク軽減策
token が誤って表示される。wrapper は GH_TOKEN を process environment に読み込み、値を echo しない。
stored Git credentials が intended token を上書きする。Git wrapper はその invocation だけ credential.helper を無効化し、askpass を使う。
元の .local/ helper を復元できない。再利用可能な wrapper logic は tracked にした。別途復元が必要なのは secret token だけ。
復旧を急いで広すぎる token を使う。task を満たす最小の repository-scoped token を使う rule は維持する。

ここで決めないこと

この変更は GitHub token を作成、拡張、公開しない。command wrapper を再現可能にするだけ。

Terminology Check

用語ここでの使い方
Credential sourcecommand が token を読む場所。原則として .env または process environment。
Wrapper scriptcredentials を保存せずに読み込む tracked command entry point。

Runtime Boundary Check

wrapper は PowerShell local helper commands。CI runtime、GitHub Actions configuration、Dev Container contract は変更しない。
