Decision Summary

この repository は、guardrails、operating rules、review records、generated control pages、CI、check scripts に集中させる。旧 app prototype、local shelf samples、Docusaurus leftovers、caches、generated local artifacts、単発 report files は削除する。

What The Owner Needs To Approve

この tree を application prototype ではなく、control system として狭く保つ方針を承認する。

Recommendation

cleanup を進める。将来の app work は、この control system を guardrail base として使い、別の意図的な app structure または repository から始める。

Risks And Mitigations

RiskMitigation
古い prototype code が削除される。owner が prototype 不要と明示。tracked files は Git history に残る。
historical records に古い paths が残る。履歴として changelog/review artifacts は残し、live rules docs と indexes を更新する。
CI や local checks が古い shelf mount を前提にする。Compose、Dev Container、CI、bootstrap、local dev scripts を generic workspace 前提に更新した。

What Is Not Decided Here

将来の application stack や app code の置き場は決めない。この変更は repository を control-system role に絞るだけ。
