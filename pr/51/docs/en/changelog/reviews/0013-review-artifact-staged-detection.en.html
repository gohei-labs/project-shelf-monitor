Review Artifact Staged Detection

Project Shelf review artifact. Status: Draft for owner review. Date: 2026-06-14.

Japanese version: 0013-review-artifact-staged-detection.ja.html

Decision Summary

Recommended Local review-artifact guardrails should inspect staged changes when no explicit changed-file list is passed.

This prevents an AI agent from staging a new review HTML file, running the local checks, and seeing a misleading none result.

Owner Approval

  • Local scripts may read Git working-tree diff and staged diff when ChangedFiles is omitted.
  • Staged new docs/reviews/*.html files must be checked by terminology and references guardrails.
  • Untracked files remain excluded from automatic detection to avoid unrelated local clutter.

Not Decided Here

  • Full documentation portal implementation.
  • Shared PowerShell helper extraction.
  • Line-ending normalization for unrelated existing files.

Before And After

Case Before After
CI pull request check Uses explicit changed files from GitHub Actions. Unchanged Still uses explicit changed files.
Local script with staged new review HTML Gap Reported changed review artifacts as none. Fixed Reads git diff --cached --name-only and checks the artifact.
Local script with unrelated untracked files Did not include them. Intentional Still does not include them.

Detection Flow

Risks And Mitigations

Risk Mitigation
False confidence Local checks could still miss untracked files. This is intentional. Agents should stage intended PR files before relying on local review-artifact checks.
Duplication Three scripts contain similar Git changed-file logic. Keep this change narrow now. Extract a shared helper later only if the behavior grows.

Terminology Check

Standard Term Meaning In This Review
staged changes Changes already added to the Git index with git add.
working-tree diff Tracked changes in the working tree that are not yet staged.
untracked files Files not yet tracked by Git. They are intentionally excluded from automatic local detection.
guardrail A repeatable check that prevents known workflow failures from recurring.
