Visible Agent Context

RecommendedGovernanceAgent Context

Decision Summary

Move project-specific control metadata out of root hidden paths and add a guardrail that fails on unapproved root dot-prefixed paths.

What The Owner Needs To Approve

Approve `agent-context/` as the shared visible agent manifest location, removal of the obsolete secret-scan allowlist, and the policy that root hidden paths require explicit tool or ecosystem reasons.

Recommendation

Proceed. This makes the repository easier to inspect and turns the naming rule into an executable check instead of a style preference.

Risks And Mitigations

RiskMitigation
Agents still look for the old hidden manifest.Add visible adapter files `AGENTS.md`, `CLAUDE.md`, and `GEMINI.md` that all point to `agent-context/agent-core-context.json`.
A future contributor adds a project-specific dotfile by habit.`scripts/guardrails/check-root-hidden-paths.ps1` fails unless the path is explicitly allowed.
Local secrets are confused with tracked control files.The policy allows ignored local runtime files such as `.env` while blocking tracked or unignored hidden control paths.

Terminology Check

Runtime Boundary Check

The runtime does not change. This adds a PowerShell guardrail to the existing local and CI check suite; it does not require a new host tool or development environment.
