Source-Driven Docusaurus Portal

Review artifact for Issue #38. This asks the owner to approve Docusaurus as a presentation layer over docs/ Markdown, not as a second source for rulebooks or specs.

What The Owner Needs To Approve

  1. Keep docs/ as the source of truth for rules, specs, and change notes.
  2. Make Docusaurus read selected Markdown files from docs/.
  3. Do not keep hand-maintained duplicate pages under docs-portal/docs/.
  4. Upload the Docusaurus build output as a CI artifact for owner inspection.

Problem Being Fixed

Previous State

Rule and spec content was copied into hand-maintained Markdown under docs-portal/docs/.

Risk

Two editable copies can drift. The owner or a future AI agent could read stale governance content.

Recommended Direction

Docusaurus reads docs/ and only handles browser presentation. Editing stays in docs/.

Boundary Map

AreaRoleAllowed hand-written content
docs/ Source of truth for AI and human-readable project memory. Rulebooks, specs, change notes, review artifacts, ADRs.
docs-portal/ Presentation layer that renders docs/ in the browser. Configuration, sidebars, styling, landing pages.
docs-portal/build/ Generated static site output. None. Do not hand-edit and do not commit.

How The Owner Can Inspect The Result
