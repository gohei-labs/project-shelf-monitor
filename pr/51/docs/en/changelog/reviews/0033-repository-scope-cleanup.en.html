Repository Scope Cleanup

RecommendedCleanupGovernance

Decision Summary

Keep this repository focused on guardrails, operating rules, review records, generated control pages, CI, and check scripts. Remove the old app prototype, local shelf samples, Docusaurus leftovers, caches, generated local artifacts, and one-off report files.

What The Owner Needs To Approve

Approve the narrower repository scope: this tree is now the control system, not the application prototype.

Recommendation

Proceed with the cleanup. Future application work should start in a separate, intentional app structure or repository using this control system as the guardrail base.

Risks And Mitigations

RiskMitigation

What Is Not Decided Here

This does not choose the future application stack or where app code should live. It only cleans this repository down to its control-system role.
