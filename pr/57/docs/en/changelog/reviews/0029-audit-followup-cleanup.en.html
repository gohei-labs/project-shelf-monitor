Audit Follow-Up Cleanup

This review asks for approval of the safe, mechanical hygiene fixes from a project-state audit, and names the two items deliberately left for an owner decision. No guardrail or product behavior changes.

Requested Approval

  1. The Docusaurus build cache (docs-portal/.docusaurus/) is now gitignored, so it cannot pollute a clone.
  2. The deprecated onBrokenMarkdownLinks option moved under markdown.hooks, clearing a Docusaurus v4 warning.
  3. The guardrail map lead copy now derives its rule count from the registry, so the prose cannot drift from the real total.
  4. The handoff brief and the reviews index no longer point at a deleted branch, old review paths, or a stale bilingual-coverage line.

Risk

Risk is low: the changes are documentation, gitignore, and generator copy, with no behavior change. Two items are intentionally deferred for an owner decision: (1) the dashboard subsystem is referenced by tracked planning docs but is untracked - commit it or mark those sections not-yet-implemented; (2) docs-portal has no committed lockfile and the build installs with --no-package-lock by design, so adopting a lockfile plus npm ci is a separate call.

Terminology Check

TermMeaning
Build cacheFiles a tool regenerates on each build (here Docusaurus' .docusaurus/); they should never be committed.
DeprecationAn option still working today but scheduled for removal in a future major version (Docusaurus v4).
Handoff briefThe short, disposable status note for the next session; it goes stale by design and is refreshed each change.
LockfileA file pinning the exact resolved dependency tree so installs reproduce; npm ci requires one.

Runtime Boundary Check

No new runtime is introduced. The Docusaurus config edit only relocates an existing option (Node and the docs-portal build are unchanged), and the guardrail map generator change is PowerShell-only. The local pre-check, the CI check job, and the Dev Container Bash lane all still run the same scripts/check.ps1. No new dependency (Node, npm, Docker) is added.
