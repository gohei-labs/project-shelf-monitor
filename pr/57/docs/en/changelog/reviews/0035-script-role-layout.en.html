Script Role Layout

RecommendedGovernanceTooling

Decision Summary

Group scripts by role while keeping the stable standard check entrypoints at scripts/check.ps1 and scripts/check.sh.

What The Owner Needs To Approve

Approve the script taxonomy: scripts/guardrails/ for gates, scripts/generate/ for monitor builders, scripts/local/ for local helpers, and scripts/setup/ for bootstrap helpers.

Recommendation

Proceed. The structure makes the guardrail body visible without inventing a competing top-level directory.

Risks And Mitigations

RiskMitigation
CI calls stale script paths.Update workflow calls and keep scripts/check.ps1 as the shared entrypoint.
Registry paths drift from real files.Update monitor/registry.json and let scripts/guardrails/check-registry.ps1 verify it.
Local helper commands move.Document the new scripts/local/ paths in README and live rule docs.

What Is Not Decided Here

This does not split local and CI into separate check suites. They intentionally share scripts/check.ps1; only local helper utilities are separated.

Terminology Check

TermUse here
Guardrail gatesExecutable validation scripts under scripts/guardrails/.
GeneratorsScripts under scripts/generate/ that write monitor/*.html.
Local helpersOwner/developer convenience scripts under scripts/local/.

Runtime Boundary Check

The runtime does not change. PowerShell remains the guardrail runtime, Bash still delegates to PowerShell, and CI continues to run the same entrypoint plus PR-context review gates.
