Primary References Guardrail Review

レビュー資料を、推測やチャット記憶ではなく、一次情報に結びつけるためのガードレール。

Approve These Decisions

今回承認してほしいこと

  • 外部事実、製品挙動、ライセンス、標準、業界用語に基づく review artifact には、見える形の References section を必須にする。
  • 一次情報を優先し、二次情報は一次情報がない場合または補足文脈としてのみ使う。
  • 外部事実に基づかない小さな artifact では、References / Not applicable と明記できるようにする。
  • 変更された review artifact が References または Not applicable を欠く場合、CIで止める。
Why

なぜ必要か

技術判断を、内輪の言葉やAIの推測ではなく、公式ドキュメント、標準、ライセンスページ、プロジェクト記録に結びつけるためです。

Limit

CIでできないこと

CIは「その資料が本当に外部事実に依存しているか」までは判断しません。References が十分かどうかは、人間レビューで確認します。

Enforce

CIで止めること

変更された docs/reviews/*.html に References section も Not applicable もない場合は、PRを失敗させます。

一次情報の優先順位

種類 例 使い方
公式情報 公式ドキュメント、vendor licensing page、standards document 現在の製品挙動、ライセンス、標準に関する判断を支える。
ソース source repository、release notes、schema definitions 実装や仕様の根拠を確認する。
プロジェクト記録 GitHub Issues、PRs、specs、ADRs、過去の review artifacts このプロジェクト内で承認済みの文脈をたどる。
二次情報 解説記事、ブログ、まとめ記事 一次情報がない場合、または補助説明としてのみ使う。
Terminology Check

標準用語チェック

  • primary source: 判断対象に最も近い一次情報。
  • secondary source: 一次情報を解説、要約、解釈した情報。
  • review artifact: オーナーが判断しやすいように作るブラウザ向け資料。
  • Not applicable: 外部事実に基づかないため References が不要であることの明示。
