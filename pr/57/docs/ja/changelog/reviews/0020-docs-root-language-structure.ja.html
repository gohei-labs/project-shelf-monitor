レビュー 0020: docs root の言語別構造化

承認してほしいこと

  1. docs 直下の top-level directory を docs/en と docs/ja だけにすること。
  2. 人間用レビュー資料を docs/en/reviews と docs/ja/reviews に移すこと。
  3. agent context manifest を docs/agent-core-context.json から .agents/agent-core-context.json に移すこと。

なぜこの変更が必要か

前回の PR scope は、仕様ファイル1組だけの移行になっていました。そのため、Docusaurus が source tree として読む docs の直下に、legacy Markdown や review folder が残っていました。これは、表示されるドキュメントの元構造が一貫しない状態です。

docs/ja/...
agent context

用語チェック

標準的な documentation 用語として、source tree、language root、manifest、review artifact、Docusaurus、Markdown を使っています。

実行境界チェック

この変更は documentation と guardrail path の整理だけです。runtime service、storage、credential、host setup は変更しません。
