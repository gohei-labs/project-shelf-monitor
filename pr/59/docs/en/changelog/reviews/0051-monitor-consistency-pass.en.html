Monitor consistency pass

This review asks for approval to polish three consistency details across the monitor pages: auto-select the first row on open, drop the redundant per-page group heading, and move the Reviews nav link to the far right.

Requested Approval

  1. Auto-select the first row on open. On wide screens, every detail page (guardrail, development rules, requirements, features) and the reviews page selects its top row on load, so the detail / reading pane is populated immediately instead of showing the empty state. Narrow screens still land on the list (the detail is an overlay, and reviews rows stay navigation links).
  2. Drop the redundant group heading. The application pages no longer render an <h2> sub-heading (Development rules / Requirements / Features / System diagram); the page title already names the content, matching the guardrail page which has no sub-heading.
  3. Move Reviews to the far right of the top nav on every page; it previously sat in the middle and was hard to scan.
