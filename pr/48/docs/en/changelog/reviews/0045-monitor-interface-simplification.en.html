Monitor Interface Simplification

RecommendedMonitorGenerated outputDesign

Decision Summary

Simplify the generated monitor pages by removing top summaries, explanatory copy, search boxes, and filter controls.

What The Owner Needs To Approve

Approve the revised monitor direction: the owner-facing monitor is a plain inspection list with grouped records and stable item detail, not a dashboard with top metrics or interactive filtering.

Recommendation

Proceed. The simplified surface better matches the owner feedback and keeps the monitor focused on direct inspection.

Risks And Mitigations

RiskMitigation
Removing search makes long lists slower to scan.Keep records grouped by pillar and category, with compact rows that preserve title, category, status, and venue.
Removing top counts hides aggregate state.Aggregate state remains available from the source registry and can return later only if it helps the owner inspect, not if it decorates the page.
The generated HTML could drift from the scripts.Regenerate monitor output and run the generator verify modes plus the repository check suite.
Mobile rows could become cramped.Use a single-column row layout on narrow screens and keep badges wrapping below the title.

What Is Not Changed

This does not change guardrail definitions, review artifact contents, CI policy, publishing credentials, or the public mirror boundary.
