監査の後続クリーンアップ

この成果物は、プロジェクト状態の監査から出た、安全で機械的な衛生修正の承認を求め、オーナー判断のため意図的に残した 2 件を明示する。ガードレールやプロダクトの挙動は変わらない。

承認のお願い（Requested Approval）

  1. Docusaurus のビルドキャッシュ（docs-portal/.docusaurus/）を gitignore し、クローンを汚さないようにした。
  2. 非推奨の onBrokenMarkdownLinks を markdown.hooks 配下へ移し、Docusaurus v4 の警告を解消。
  3. ガードレール地図のリード文がルール数をレジストリから導出するので、文言が実数からずれない。
  4. 引継ぎブリーフとレビュー索引が、削除済みブランチ・古いレビューパス・古いカバレッジ記述を指さなくなった。

リスク（Risk）

リスクは低い：変更はドキュメント・gitignore・生成器の文言で、挙動変更はない。2 件を意図的に先送りする：(1) ダッシュボード一式は追跡済みの計画ドキュメントから参照されるのに未追跡——コミットするか、当該節を未実装と明記するか。(2) docs-portal にコミット済み lockfile が無く、ビルドは設計上 --no-package-lock で install するため、lockfile ＋ npm ci の採用は別の判断。

用語チェック（Terminology Check / 標準用語チェック）

用語意味
ビルドキャッシュツールがビルドごとに再生成するファイル（ここでは Docusaurus の .docusaurus/）。コミットしてはならない。
非推奨（deprecation）今は動くが将来のメジャー（Docusaurus v4）で削除予定のオプション。
引継ぎブリーフ次セッション向けの短い使い捨ての現在地。設計上古くなり、変更ごとに更新する。
lockfile解決済み依存ツリーを固定し install を再現させるファイル。npm ci はこれを要する。

ランタイム境界チェック（Runtime Boundary Check）

新たなランタイムは導入しない。Docusaurus 設定の編集は既存オプションの再配置のみ（Node とポータルのビルドは不変）で、ガードレール地図生成器の変更は PowerShell に閉じる。ローカル事前チェック、CI の check ジョブ、Dev Container の Bash レーンは引き続き同じ scripts/check.ps1 を実行する。新しい依存（Node・npm・Docker）は追加しない。
