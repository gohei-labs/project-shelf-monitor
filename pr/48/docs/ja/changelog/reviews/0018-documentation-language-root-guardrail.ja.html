ドキュメント言語別ルートガードレール

このレビューでは、Markdown ドキュメントの配置先を docs/en/ と docs/ja/ に固定することを承認してほしいです。

承認してほしいこと

  1. 新しい Markdown ドキュメントは、必ず docs/en/ と docs/ja/ の両方に作る。
  2. 既存の旧配置 Markdown は、正式な構造ではなく移行バックログとして扱う。
  3. 移行バックログは docs/legacy-language-root-allowlist.txt に明示する。
  4. 言語別ルートの外に新しい Markdown が追加された場合、CI で止める。

構造

場所ルール
docs/en/英語版 Markdown ドキュメントを置く。
docs/ja/同じ相対パスで日本語版 Markdown ドキュメントを置く。
docs/reviews/オーナー承認用の英語版・日本語版 HTML レビュー資料を置く。
その他の docs/ 配下 Markdown一時的な移行バックログだけを許可する。

リスク

この変更だけでは、旧配置の文書をすべて翻訳・移動するところまでは完了しません。新しいズレを防ぎ、残っている移行作業を見えるようにします。

標準用語チェック

用語意味
言語別ルートdocs/en/ や docs/ja/ のように、言語ごとに分けたドキュメント配置先。
移行バックログ目標構造へ移すまで、一時的に許可している既存ファイル。
allowlistCI が確認できる、一時的な例外の明示リスト。

実行環境境界チェック

この変更は、リポジトリ内のドキュメント構造と CI チェックの変更です。Windows ホストの npm、Docker Desktop、アプリサーバーの起動は必要ありません。
