レビュー資料テンプレート

この資料は、今後のオーナー向けレビュー資料を単一の編集的テンプレートに統一し、あわせてランタイム境界チェックを廃止することの承認を求めます。

承認のお願い（Requested Approval）

  1. 編集的レイアウト（ニアモノクロ＋ライム単一アクセント、Space Grotesk＋Inter）を、オーナー向けレビュー資料の正式テンプレートにする。メタ情報（番号・状態・日付）は最上部の帯に集約する。
  2. 固定セクション順を定める：承認のお願い → 変更内容 → リスク → 決めないこと → 資料チェック（用語・一次情報）。「資料チェック」は、用語と一次情報を著者が確認したことを示すチェックリストとする。
  3. ランタイム境界チェックをテンプレートから外し、対応する CI ゲート（review-artifact-runtime-boundary / LLM-GR-006）を無効化する。必要になれば再び強制に戻す。

リスク（Risk）

形式の刷新は過去の青／琥珀 callout 形式（0024–0027）から離れるが、承認アンカーである「承認のお願い」を先頭・最重要に保ち、ライムを「決めるべき箇所」だけに限定して判断のしやすさを維持する。ランタイム境界チェックの廃止により、ホストやツール前提の取り違え（PR #36 型の事故）を CI が機械的に捕まえる力は弱まる。必要になれば再強制でき、権威ある環境は引き続き Dev Container と Docker Compose、ローカルも CI も同じ check.ps1 を実行する。外部参照は表示時の Google Fonts のみで、遮断時はシステムフォントへ劣化する。

決めないこと（Not Decided Here）

  • 既存 47 本のレビュー資料の遡及修正（統一は今後分のみ。オーナー確認済みで Issue 化しない）。
  • rules doc（review-artifacts.md）の Content Model とテンプレートの整合（別途。オーナー確認済みで Issue 化しない）。
  • ランタイム境界ガードレールを将来いつ再強制するか（必要が生じた時点で判断）。
