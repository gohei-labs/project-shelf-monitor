Decision Summary

Recommended C: drive cleanup を先に行ったうえで、Docker Desktop + Docker Compose をこのPCのローカル開発入口として使います。

この資料は、Docker Desktop の導入そのものと、`scripts/run-local-dev.ps1` で再実行できる smoke test の範囲を承認してもらうためのものです。

Owner Approval

  • Docker Desktop をこのPCの最初の Docker-compatible runtime として使う。
  • `scripts/run-local-dev.ps1` をローカル Docker 検証の標準入口にする。
  • `/shelf` read-only mount check を標準検証に含める。
  • Dev Container CLI は optional のままにし、VS Code Dev Containers は人間の作業導線として扱う。

Not Approved Here

  • アプリ本体の backend/frontend 実装。
  • `/shelf` への write access。
  • Docker storage を C: 以外へ移す判断。
  • 将来の商用利用時の Docker Desktop license 最終判断。

Technical Adjustments

Adjustment Reason
`compose.yaml` sets `working_dir: /workspace` and `user: vscode`. Compose 直実行でも Dev Container に近い作業条件にします。
`bootstrap.sh --quick` does not require Docker CLI inside the container. Docker CLI は host preflight で確認します。workspace container の quick bootstrap は依存を増やしません。
Dev Container image registers `/workspace` as Git `safe.directory`. Windows bind mount の ownership 差分で Git hygiene check が止まるのを防ぎます。

Risks And Follow-Up

Noise CRLF/LF warnings appeared for two existing files. check failure ではありません。運用上うるさい場合は line-ending normalization を別 issue にします。
