ローカル事前チェック証跡

Project Shelf レビュー資料。ステータス: オーナーレビュー用ドラフト。日付: 2026-06-14。

英語版: 0014-local-precheck-evidence.en.html

判断の概要

推奨ローカル事前チェックを実行したら、その結果を JSON 証跡として .local/check-runs/ に保存します。

これにより、PR本文や Codex review record で「チェックした」と書くだけでなく、いつ、どのブランチで、どのコマンドを実行し、何が通ったかを確認できるようにします。

承認してほしいこと

  • scripts/save-local-precheck.ps1 を、PR前のローカル事前チェック証跡を作る標準入口にする。
  • 証跡は既定で .local/check-runs/ に保存し、Gitには commit しない。
  • PR本文と Codex review record では、証跡ファイルの path と要約を参照する。

ここでは決めないこと

  • CI artifact としての保存。
  • 署名付き証跡や改ざん防止の仕組み。
  • 大きな test-reporting system。

保存される内容

項目 目的
timestamp いつ実行したかを確認する。
branch / head commit どの作業状態で実行したかを確認する。
base ref / changed files どの基準ブランチとの差分を検査対象にしたかを確認する。
working tree state 未コミット変更がある状態で実行したかを確認する。
command list / pass-fail status どのチェックを実行し、どれが通ったかを確認する。
short output summary 詳細ログを全部読まなくても、結果の雰囲気を把握する。

実行の流れ

PR本文下書きを用意
->
save-local-precheck.ps1
->
origin/main...HEAD
とローカル差分を収集
->
チェック実行
->
JSON証跡を保存
->
PR本文で参照

リスクと対応

リスク 対応
ローカル証跡証跡は Git-ignored なので、外部の不変な監査記録ではない。 今回は「PR前に何を実行したかを見える化する」小さなガードレールに留めます。より強い監査性が必要なら CI artifact を別 issue で検討します。
出力の肥大化コマンド出力を全部保存すると読みにくくなる。 証跡には短い output summary を残します。詳細な失敗調査が必要な場合は、再実行またはCIログを使います。

用語チェック

用語 意味
local pre-check PR作成前またはPR更新前に、ローカルで実行する確認。
evidence チェックを実行したことを後から確認するための証跡。
machine-readable 人間だけでなく、スクリプトやAIエージェントも読める形式。今回は JSON。
base ref 差分を計算するときの基準。既定では origin/main。
